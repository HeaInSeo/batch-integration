# ADR-001 Canonical Document Policy

상태: 승인

## 결정

- 저장소 내부 상세 설계의 원본은 각 저장소에 둔다.
- `batch-integration/`은 cross-repo 일정, 계약, 운영 규칙의 원본이다.
- 원본 계획 문서는 `plans/original/`에 보존한다.
- 수정 계획과 delta 문서는 `plans/revised/`에 둔다.

## 이유

- 같은 내용을 여러 곳에서 원본처럼 관리하면 빠르게 드리프트가 발생한다.
- 이번 작업은 저장소별 설계보다 저장소 간 의존성과 순서 관리가 더 중요하다.

## 결과

- 저장소 내부 상세 변경은 각 저장소 문서에서 추적한다.
- 이 허브에서는 일정, 경계, 게이트, 계약 변경만 관리한다.
